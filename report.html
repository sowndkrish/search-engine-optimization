Project Requirement

The project requirement is to build a search engine for academic papers which includes all the features of an information retrieval system. There are various features such as PageRank, Classification, Clustering, Doc2vec, Word2vec, web crawling etc. Some of the improvements which could be done is to include sorting by title, pagination, more datasets, context around the display etc. The search engine is developed by Apache Lucene in java and hosted with the help of Apache Tomcat.

Implementation

The data is consists of CiteSeer academic papers which is semi-structured in the txt format.

Apache Tomcat

Download the latest version of Apache Tomcat to host the search engine. The apache tomcat runs at 8080 port by default.

  • bin: contains the binaries; and startup script.
  • conf: contains the system-wide configuration files, such as server.xml, web.xml, context.xml, and tomcat-users.xml.
  • lib: contains the Tomcat's system-wide JAR files, accessible by all webapps.
  • logs: contains Tomcat's log files and error messages.
  • webapps: contains the webapps to be deployed. We can place the WAR (Webapp Archive) file for deployment here.
  • work: Tomcat's working directory used by JSP, for JSP-to-Servlet conversion.

Configure Tomcat

In the tomcat-users.xml file add the various user rolenames such as manager-status,manager-gui,manager-script and for admin, manager set the password. This password must be entered while accessing apache tomcat at localhost:8080. After entering this url in the browser, if the browser displays the default page then the server is up.

Configure Eclipse

In the web.xml file under WEB-INF folder, provide the servlet mapping to the class java file. We can also provide the welcome file to display when the tomcat loads. Once the project is created and coded, export the war file and deploy in the apache folder and start the server through 'monitor tomcat' interface.
